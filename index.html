Reproducible Papers Manifesto

If you have written a Computer Science, answer the following questions:

1. Do you want people to read your paper and build on your research?

2. Do you want to get more citations and allow others to easily use your work?

3. Do you want others to trust your results?

4. Do you want to have access to all the code and data of other researchers?

5. Do you want to be able to verify their results and see how they did it?

Because this is the manifesto for you!

You read a great paper and have an interesting idea for a new paper that builds on the results of that paper. Unfortunately, they haven't released their code and their data, so you have to do all the work again. What a waste of time! Others will waste time doing the work you already did, and you'll waste time doing work others already did.

  • Wouldn't it be nice to peek at the Source Code of that latest machine learning paper to understand it better?
  • Wouldn't it be great to have access to that Amazing Dataset you saw in That Paper for your own paper?
  • Wouldn't it be pleaseant to have a nice Notebook or Script recreating the results you read?

It would be amazing if others did it! To make this a reality, we must do it too. If you already spent time doing your discovery, why keep it to yourself? So let's share our code and data openly and make everyone's lives easier.

We need to fix this. We need new ways to share our research and to do Open Science, because our current system is broken and makes our lives more difficult.

Principles - How to Share Research

We want other people to be able to reproduce your results on their devices and to build on them. How can you help them?

1. Code and data must be easily downloadable

If you want other's code and data, you must share yours.

The word easily is there for a reason! This means no paywall, no complicated procedure: clicking a link downloads the files of the whole project. If others have to respect certain constraints - no commercial use, privacy regulations, or others - you can of course ask for consent first. However, keep this to a minimum.

2. The code must execute and reproduce all the results shown in the paper

If the code doesn't re-create the results of your paper, you're reproducing your results.

3. A simple script should produce all the results - no 23 step process

We are humans and humans are lazy. Make it as easy as possible to reproduce your results.

4. Prefer modern communication tools - avoid fax, avoid snail mail

Unless absolutely required, e-mail or web tools are much faster and easier for everyone.

5. Use standard and widely used tools

For example, if you want to share code, using Github or Gitlab is easier for everyone. It's also a lot more user friendly, you don't have to worry about the hosting and it makes your paper more discoverable.

Checklist

1. Is all of your code available?

2. Is all of your data available?

3. Is there a runnable script?

4. Does the script produce all your results?

5. Can you download everything with one click?

6. Do you use widely used tools?

7. Can you be contacted by e-mail or the web?


The easiest way to do it

You might be thinking "This is very nice, but I don't have the patience or time to do this. I'm a busy researcher and have other things to do. This will be too much work".

And you'd be completely wrong! It's actually really easy.

Here's a simple step-by-step guide to publishing your results:

How to share easily

1. Make a script that produces all your results

2. Create a Github account

3. Create a new repository

4. Click on the Upload Files button

5. Add your files

6. You're done!

You just shared your code and data with the world for free. You have very-high quality hosting and it's easily accessible by anybody. And it barely took 5 minutes.
